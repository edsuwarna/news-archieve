# Kubernetes Security News — 2026-05-31

## 🔴 Critical

### 🔴 TeamPCP Supply Chain Attacks — Worm-Like Cloud Campaigns
TeamPCP (aka DeadCatx3, PCPcat) remains the most active threat actor targeting the Kubernetes ecosystem. Multiple incidents:

- **Trivy Supply Chain Compromise (CVE-2026-33634, CVSS 9.4)** — Aqua Security's Trivy vulnerability scanner was compromised. Threat actors pushed trojanized versions (0.69.4–0.69.6) to Docker Hub containing a credential stealer. The breach spread to Checkmarx GitHub Actions and later to LiteLLM (versions 1.82.7–1.82.8) which included a K8s lateral movement toolkit deploying privileged pods.
- **PCPJack Credential Stealer** — A worm-like framework targeting Docker, Kubernetes, Redis, MongoDB, RayML, and web apps. Exploits 5 CVEs to harvest credentials from cloud/container services and spread laterally.
- **Worm infrastructure campaign (Dec 2025–present)** — Exploited exposed Docker APIs, K8s clusters, Ray dashboards, Redis servers, and React2Shell (CVE-2025-55182, CVSS 10.0) to build criminal infrastructure.

### 🔴 GitHub Employee Device Breach (May 20, 2026)
GitHub confirmed unauthorized access to 3,800+ internal repos after TeamPCP breached an employee device. While no customer data impact was reported, the source code exposure poses downstream supply chain risks.

## 🟡 High/Medium CVEs

### 🟡 CVE-2026-4342 — ingress-nginx Comment-Based Config Injection (CVSS 8.8, HIGH)
- **Date:** Mar 20, 2026
- **Description:** Ingress annotations can be used to inject nginx configuration, leading to RCE in the ingress-nginx controller context and disclosure of cluster-wide Secrets.
- **Affected:** ingress-nginx < v1.13.9, < v1.14.5, < v1.15.1
- **Fixed:** v1.13.9, v1.14.5, v1.15.1
- **Reported by:** wooseokdotkim

### 🟡 CVE-2026-3288 — ingress-nginx rewrite-target Config Injection (CVSS 8.8, HIGH)
- **Date:** Mar 10, 2026
- **Description:** The `nginx.ingress.kubernetes.io/rewrite-target` annotation can be used to inject nginx configuration, leading to RCE and Secret disclosure.
- **Affected:** ingress-nginx < 1.13.8, < 1.14.4, < 1.15.0
- **Fixed:** 1.13.8, 1.14.4, 1.15.0
- **Reported by:** Kai Aizen
- **Mitigation:** Admission control can block rewrite-target annotation usage before patching.

### 🟡 Multiple ingress-nginx Issues (Feb 2, 2026) — 4 CVEs (CVSS 8.8, HIGH)
- CVE-2026-1580, CVE-2026-24512, CVE-2026-24513, CVE-2026-24514
- **Affected:** ingress-nginx < v1.13.7, < v1.14.3
- **Fixed:** v1.13.7, v1.14.3

### 🟡 CVE-2026-3865 — CSI Driver for SMB Path Traversal (CVSS 6.5, MEDIUM)
- **Date:** Apr 11, 2026
- **Description:** Insufficient `subDir` validation allows path traversal; malicious PersistentVolume with `../` sequences can delete/modify unintended directories on the SMB server.
- **Affected:** CSI Driver for SMB < v1.20.1
- **Fixed:** v1.20.1
- **Reported by:** Shaul Ben Hai (SentinelOne)

### 🟡 CVE-2026-3864 — CSI Driver for NFS Path Traversal (CVSS 6.5, MEDIUM)
- **Date:** Mar 17, 2026
- **Description:** Same subDir path traversal issue as SMB variant but for NFS CSI driver.
- **Affected:** CSI Driver for NFS
- **Reported by:** (Similar to CVE-2026-3865)

### 🟡 Docker CVE-2026-34040 — AuthZ Bypass (CVSS 8.8, HIGH)
- **Date:** Apr 7, 2026
- **Description:** Incomplete fix for CVE-2024-41110. Attackers can bypass authorization plugins via specially crafted API requests that omit request bodies. Affects Kubernetes environments using Docker with authorization plugins.

### 🟡 LiteLLM Supply Chain Compromise (Mar 24, 2026)
TeamPCP backdoored LiteLLM versions 1.82.7–1.82.8 via Trivy CI/CD compromise. Payload included a 3-stage attack: credential harvester (SSH keys, cloud creds, K8s secrets, crypto wallets), K8s lateral movement toolkit (privileged pods to every node), and persistent systemd backdoor.

### 🌀 K8s Security Trends
- **ingress-nginx under sustained attack:** 3 separate advisories in Q1–Q2 2026 with CVSS 8.8, all involving annotation-based config injection. Multiple coordinated fixes across 1.13.x, 1.14.x, and 1.15.x branches. **Strong recommendation:** Keep ingress-nginx up to date and use admission controllers to block untrusted annotations.
- **CSI driver path traversal:** SMB and NFS CSI drivers both had the same subDir validation flaw, discovered by SentinelOne. Highlights need for input sanitization in storage drivers.
- **Supply chain attacks escalating:** The Trivy → Checkmarx → LiteLLM chain demonstrates that CI/CD pipeline compromises can cascade across the ecosystem rapidly. TeamPCP's credential stealer specifically targets `kubeconfig` files and cloud provider credentials.
