## 🔴 CRITICAL / HIGH SEVERITY ADVISORIES

### 1. CVE-2026-3865 — CSI Driver for SMB Path Traversal (Apr 11)
**Severity:** Medium (CVSS 6.5)
**Component:** CSI Driver for SMB (`smb.csi.k8s.io`)
**Vulnerability:** Insufficient validation of the `subDir` parameter in volume identifiers allows path traversal. A malicious user with PV creation privileges can craft a `volumeHandle` containing `../` sequences. During volume deletion cleanup, the driver may operate on unintended directories on the SMB server.
**Impact:** Deletion/modification of directories outside the intended managed subdirectory within the SMB export.
**Affected:** All versions < v1.20.1
**Fix:** Upgrade to >= v1.20.1
**Mitigation:** Restrict PV creation to trusted admins; review SMB exports.
**Reporter:** Shaul Ben Hai (SentinelOne)
**Reference:** https://github.com/kubernetes/kubernetes/issues/138319

### 2. CVE-2026-4342 — ingress-nginx Comment-Based Config Injection (Mar 20)
**Severity:** HIGH (CVSS 8.8)
**Component:** ingress-nginx
**Vulnerability:** A combination of Ingress annotations can be used to inject configuration into nginx, leading to arbitrary code execution in the ingress-nginx controller context and disclosure of Secrets accessible to the controller (by default, all Secrets cluster-wide).
**Affected:** ingress-nginx < v1.13.9, < v1.14.5, < v1.15.1
**Fix:** Upgrade to v1.13.9 / v1.14.5 / v1.15.1
**Detection:** Suspicious data in `rules.http.paths.path` field of Ingress resources.
**Reporter:** wooseokdotkim
**Reference:** https://github.com/kubernetes/kubernetes/issues/137893

**Fix:** Upgrade to >= v4.13.1
**Mitigation:** Restrict PV creation to trusted admins.
**Reporter:** Shaul Ben Hai (SentinelOne)

### 4. CVE-2026-3288 — ingress-nginx rewrite-target Config Injection (Mar 10)
**Severity:** HIGH (CVSS 8.8)
**Component:** ingress-nginx
**Vulnerability:** The `nginx.ingress.kubernetes.io/rewrite-target` annotation can be used to inject configuration into nginx, enabling arbitrary code execution and cluster-wide Secret disclosure.
**Affected:** ingress-nginx < v1.13.8, < v1.14.4, < v1.15.0
**Fix:** Upgrade to v1.13.8 / v1.14.4 / v1.15.0
**Pre-upgrade Mitigation:** Use admission control to block the `rewrite-target` annotation.
**Reporter:** Kai Aizen
**Reference:** https://github.com/kubernetes/kubernetes/issues/137560

## 🔥 ECOSYSTEM THREAT LANDSCAPE

### TeamPCP / Supply Chain Attacks
A persistent threat actor (TeamPCP/PCPcat/DeadCatx3) has been systematically targeting cloud-native infrastructure:

- **GitHub Breach (May 20):** Employee device compromise led to exfiltration of 3,800+ internal repos. Source code listed for sale at $50K+.
- **PCPJack Credential Stealer (May 7):** Worm-like framework exploiting 5 CVEs to harvest credentials from Docker, Kubernetes, Redis, MongoDB, and RayML. Spreads laterally within compromised networks.
- **LiteLLM Backdoor (Mar 24):** Versions 1.82.7–1.82.8 compromised with 3-stage payload: (1) credential harvester (SSH keys, cloud creds, K8s secrets, crypto wallets, .env), (2) K8s lateral movement toolkit deploying privileged pods to every node, (3) persistent systemd backdoor.
- **Checkmarx GitHub Actions (Mar 24):** CI/CD compromise via stolen credentials from Trivy attack.
- **Trivy Supply Chain Attack (CVE-2026-33634, CVSS 9.4):** Compromised credentials used to push infostealer via Docker Hub images.
- **TeamPCP Worm (Feb 9):** Massive campaign exploiting exposed Docker APIs, K8s clusters, Ray dashboards, and Redis servers.

### Other Threats
- **Docker CVE-2026-34040 (Apr 7, CVSS 8.8):** AuthZ bypass via specially-crafted API requests. Incomplete fix for CVE-2024-41110.
- **Fluent Bit Flaws (Nov 2025):** 5 vulnerabilities chained for RCE and infrastructure compromise.
- **UNC4899 (Mar 2025):** North Korean threat actor targeting crypto firms via social engineering, cloud compromise, and container breakout.

## 🛡️ BEST PRACTICES & RECOMMENDATIONS

### Immediate Actions
1. **Upgrade ingress-nginx** to the latest version (>= v1.15.1) — five distinct HIGH severity advisories in 2026 alone
2. **Upgrade CSI Drivers** — SMB to v1.20.1+, NFS to v4.13.1+
3. **Restrict PersistentVolume creation** to trusted administrators only
4. **Review ingress-nginx annotations** — block `rewrite-target` annotation via admission control if upgrade is delayed

### Ongoing Hardening
1. **Pod Security Admission** — Enforce baseline/restricted Pod Security Standards
2. **Network Policies** — Implement least-privilege network segmentation
3. **RBAC** — Follow least-privilege principle; audit `cluster-admin` bindings
4. **Secrets management** — Use external secrets providers (e.g., External Secrets Operator)
5. **Admission Controllers** — Gatekeeper or Kyverno for policy enforcement
6. **Image scanning** — Regular vulnerability scanning with Trivy or similar

### Supply Chain Security
1. **Verify image signatures** — Use cosign for container image verification
2. **SBOM** — Maintain Software Bills of Materials for all deployed images
3. **CI/CD pipeline security** — Audit GitHub Actions and CI workflows for credential exposure
4. **Monitor for malicious packages** — Use tools like Endor Labs, Socket, or JFrog Xray

### Monitoring & Detection
1. **Audit logging** — Enable Kubernetes audit logs and ship to SIEM
2. **K8s security tooling** — Falco, Kube-bench, Kube-hunter
3. **Monitor for unusual PV creation** — Look for traversal sequences (`../`) in volume handles
4. **Watch for privileged pod deployments** — Especially across multiple nodes (TeamPCP pattern)
